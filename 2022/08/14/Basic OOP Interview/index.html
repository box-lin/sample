Why do we need OOP

  • OOP help us to think in terms of real world objects. We are able to think pieces then come up with complex system. A bottom-up design approach.
  • Lot of object oriented patterns we can use to help us achieve high cohesion low coupling state, supports the general goals of high readability and maintainability.

What are the important pillars in OOP?

  • Abstration: Show only what is neccessary
  • Polymorphism: Object acts differently under different condition
  • Inheritance: Parent child relationship
  • Encapsulation: Hide complexity

What is a class and object?

  • Class is a bluprint, object is an instance of the class.

Abstraction vs Encapsulation (thought process)

  • Abstraction show what is neccessary, happens in design phase, determine what has to shown public.
  • Encapsulation hide complexity, happens in execution phase, implements abstraction.
  • Access identifier to implement these thought processes

Explain the virtual keyword in C#?

  • Virtual keyword helps us to define some logic in the parent class which can be overriden in the child class

What is differences between overloading and overriding?

  • Method overloading means same method names with different signature in the same class.
  • Overriding happens method with virtual keyword in parent class and overriding that method using override keyword in child class.
  • The overridden base method must be virtual , abstract , or override in C#.

What is polymorphism?

  • An ability for object to act differently under different condition.
  • We can’t implement polymorphism without inheritance.

Static vs Dynamic Polymorphism in C#

  • Static polymorphism is implemented by Method overloading
  • Dynamic polymorphism is implemented by method overriding

Operator overloading

  • helps us to redefine additional functionalities for plus, minus, multi, and addition.

Why do we need abstract class?

  • Abstract class is a partially defiend parent class.
  • We can’t create instance of a abstract class.
  • Some implementation is defined and some implementation left for child classes to define.
  • All abstract method must be implemented in child class.

Are abstract method virtual?

  • YES, allowing overriding.

Explain interfaces and why do we need it

  • Interface is a contract, by having contract we have better control on impact analysis, change management and breaking changes.
  • All methods, properties inside interface are public.

If you want to change interface whats the best practice?

  • We can inherited multiple inheriences, helps to add new methods without affecting the old interfaces.

Differences between abstract class and interfaces

  • Abstract class can have fields whereas an interface don’t have.
  • An interface can only inherit from another interface it cannot inherit from abstract class whereas an abstract class can inherit from another abstract class and an interface.
    Note: there used to be another difference that an interface cannot have implementation and access modifiers but C# 8.0 changed that.Abstract class members are private by default whereas interface members are public by default.
