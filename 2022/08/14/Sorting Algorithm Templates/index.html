Quick Sort

  1. Pick first number and run partition
  2. Run quicksort on left
  3. Run quicksort on right

Recurrence formula: $ f(n) = f(r-1) + f(n-r) + an$

  • Avg Time: $O(nlogn)$
  • Worst Time: $O(n^2)$
    • Why? happens to use quicksort to sort a sorted items (for quicksort that pick first item as pivot)
    • In this case we have to partition n times, each partion costs $O(n)$, $nO(n) = O(n^2)$
  • Python Implementation

Merge Sort

  1. Pick the mid
  2. Run merge sort on left
  3. Run merge sort on right
  4. Merge left and right

Recurrence formula: $ f(n) = f(n/2) + f(n/2) + an$

  • Avg Time: $O(nlogn)$
  • Worst Time: $O(nlogn)$
  • Python Implementation

Insertion Sort

  1. Run insertion sort on A[1… n-1]
  2. Insert A[n] into sorted A[1…n-1]

Recurrence formula: $ f(n) = f(n-1) + O(n)$

  • Worst Time: $O(n^2)$
  • Python Implementation
