Basic Database Interview Questions

2022-08-15
Database

Explain Normalization

  • A database design technique to remove redundant data

How do we implement normalization

  • Normalizaiton is implemented by splitting tables in to two, one for reference data (master table) and the other for transaction data.

What is denormalization

  • A database design technique to improve search performance, we merge tables.

Explain OLTP vs OLAP

  • OLTP: Online Transaction Processing
  • OLAP: Online Analytical Processing
  • OLTP Normlization avoids redundancy and we follow normalization design
  • OLAP Denormalization improve search performance and we follow denormalization design
  • Two databases

Explain 1st,2nd,and 3rd normal form

  • 1st normal form: A table when the columns have atomic values. It should not have repeating groups. (all column should only have atomic value)
  • 2nd normal form: 1st normal form should be satisfied. All non-key columns should be fully dependent on the primary key
  • 3rd normal form: All 1st and 2nd normal form should be satisfied. No transient depedency should be present. (cannot have non-key column depend on another non-key column)

Difference between primary key vs Unique key?

  • Primary key can’t be null, unique can be null
  • In table table, we can only have one primary key (single or combined) but multiple fields for unique key.

Differences between char vs varchar

  • char is fixed length
  • varchar is variable length

Differentiate between Char vs Nchar

  • nchar = non english character, 1 character 2 bytes
  • char, 1 character 1 bytes

What is the use of index

  • Increases search performance

What are the two types of indexes?

  • Clustered vs Non-Clustered Index

Differences between clustered vs non-clustered index

  • Clustered index: leaf nodes point to actual data
  • Non clustered index leaf node take help of clustered index to locate the actual data
  • Only one clustered index in a table
  • Many clustered index in a table

Function vs Stored Procedures

  • Function: Computation, only selects allowed.
  • Stored procedures: mini batch program, can change the enviorment.

What are types of trigger

  • Ater trigger: execute after event happened
  • Instead of trigger: to prevent some operations, instead of the event the logic is executed

What is transaction

  • helps to implement series of activity treat as one single unit.

What is inner joins

  • Inner join selects matching records from both tables

What is left join

  • All data from left table selected and only matching records from right table

What is full outer join

  • All matching and unmatching records from both left and right table are selected

What is cross join

  • Cross join is cartesian. Every record of one table is joined with other table records.

View
Views can represent a subset of the data contained in a table
